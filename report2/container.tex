\section{Module \texttt{container.c}}

Les conteneurs sont essentiels pour structurer l'interface. Sans eux, GTK placerait tous les widgets au même endroit. Ce module résout ce problème en abstrayant les logiques de placement via des configurations simples. L'objectif est de permettre au développeur de penser en termes de "lignes", "colonnes" ou "empilement de vues", sans se soucier de l'ordre exact des appels d'initialisation GTK.

\subsection{Fonction \texttt{create\_grid}}

La grille bidimensionnelle nécessite un réglage minutieux de l'homogénéité et de l'espacement sur deux axes. Cette fonction encapsule ces appels en extrayant simultanément les données pour les lignes et les colonnes de la structure \texttt{grid\_config}, offrant ainsi un point d'entrée unique pour la mise en page d'un formulaire complexe.

\begin{lstlisting}[language=C]
GtkWidget *create_grid(const grid_config *config) {
    GtkWidget *grid = gtk_grid_new();
    if (config == NULL) {
        return grid;
    }

    gtk_grid_set_column_spacing(GTK_GRID(grid), config->column_spacing);
    gtk_grid_set_row_spacing(GTK_GRID(grid), config->row_spacing);
    gtk_grid_set_column_homogeneous(GTK_GRID(grid), config->homogeneous_columns);
    gtk_grid_set_row_homogeneous(GTK_GRID(grid), config->homogeneous_rows);
    apply_widget_style(grid, &config->style);
    return grid;
}
\end{lstlisting}

\subsection{Fonction \texttt{create\_box}}

La boîte linéaire est le conteneur le plus utilisé. La fonction utilise l'opérateur ternaire lors de l'instanciation de base pour pallier au fait qu'un pointeur \texttt{NULL} est autorisé en entrée, garantissant ainsi que l'application ne plantera pas si le développeur omet totalement la configuration (elle générera une boîte verticale par défaut).

\begin{lstlisting}[language=C]
GtkWidget *create_box(const box_config *config) {
    GtkWidget *box = gtk_box_new(
        config ? config->orientation : GTK_ORIENTATION_VERTICAL,
        config ? config->spacing : 0
    );
    if (config == NULL) {
        return box;
    }

    gtk_box_set_homogeneous(GTK_BOX(box), config->homogeneous);
    apply_widget_style(box, &config->style);
    return box;
}
\end{lstlisting}

\paragraph{Macros associées :}
L'utilisation de macros comme \texttt{BOX\_H(sp)} et \texttt{BOX\_V(sp)} démontre la puissance expressive des littéraux composés pour la définition des structures de mise en page. Plutôt que de déclarer une variable nommée de type \texttt{box\_config} en amont, la macro injecte directement la structure anonyme instanciée au sein même de l'appel de fonction. L'initialisateur désigné garantit que l'orientation est correctement définie sur le bon axe avec l'espacement requis, tout en laissant implicitement les autres propriétés (comme le style global ou l'homogénéité) à leurs valeurs nulles par défaut (sûres et neutres grâce à la garantie de mise à zéro du C99). Cela réduit drastiquement la verbosité du code de composition d'interface.

\begin{lstlisting}[language=C]
#define ORIENT_HORIZ GTK_ORIENTATION_HORIZONTAL
#define ORIENT_VERT  GTK_ORIENTATION_VERTICAL

#define BOX_H(sp) (box_config){.orientation = ORIENT_HORIZ, .spacing = sp}
#define BOX_V(sp) (box_config){.orientation = ORIENT_VERT, .spacing = sp}
\end{lstlisting}

\subsection{Fonction \texttt{create\_stack}}

Dans le développement d'interfaces modernes, le principe du basculement de vues (navigation à onglets, assistants étape par étape) est omniprésent. Le widget \texttt{GtkStack} est le composant de référence pour cette tâche, et la fonction \texttt{create\_stack} vise à simplifier son intégration au maximum.

L'implémentation de base d'une \texttt{GtkStack} nécessite souvent de multiples appels pour configurer non seulement le comportement du conteneur, mais aussi l'expérience visuelle globale. Notre constructeur adresse directement ces problématiques en extrayant de la structure \texttt{stack\_config} les paramètres cruciaux de l'animation. Ainsi, l'appel à \texttt{gtk\_stack\_set\_transition\_type} définit la nature cinétique du changement de page (fondu, glissement), tandis que \texttt{gtk\_stack\_set\_transition\_duration} paramètre le rythme exact de cette animation en millisecondes. 

Par ailleurs, la fonction intègre intelligemment le contrôle dimensionnel via \texttt{vhomogeneous} et \texttt{hhomogeneous}. Ces réglages évitent les redimensionnements brusques et inesthétiques de la fenêtre globale lorsque l'utilisateur bascule entre une page très dense et une page presque vide, forçant le conteneur à adopter l'empreinte spatiale la plus large requise par ses enfants.

\begin{lstlisting}[language=C]
GtkWidget *create_stack(const stack_config *config) {
    GtkWidget *stack = gtk_stack_new();
    if (config == NULL) {
        return stack;
    }

    gtk_stack_set_transition_type(GTK_STACK(stack), config->transition);
    gtk_stack_set_transition_duration(GTK_STACK(stack), config->duration_ms);
    gtk_stack_set_vhomogeneous(GTK_STACK(stack), config->vhomogeneous);
    gtk_stack_set_hhomogeneous(GTK_STACK(stack), config->hhomogeneous);
    apply_widget_style(stack, &config->style);
    return stack;
}
\end{lstlisting}

\paragraph{Macros associées :}
Les macros de transition et de vitesse permettent de configurer le widget \texttt{GtkStack} avec des constantes lisibles et réutilisables.

\begin{lstlisting}[language=C]
#define TRANSITION_NONE         GTK_STACK_TRANSITION_TYPE_NONE
#define TRANSITION_CROSSFADE    GTK_STACK_TRANSITION_TYPE_CROSSFADE
#define TRANSITION_SLIDE_LEFT   GTK_STACK_TRANSITION_TYPE_SLIDE_LEFT
#define TRANSITION_SLIDE_RIGHT  GTK_STACK_TRANSITION_TYPE_SLIDE_RIGHT

#define TRANSITION_FAST   250
#define TRANSITION_NORMAL 400
#define TRANSITION_SLOW   800
\end{lstlisting}

\subsection{Helpers complémentaires : \texttt{create\_stack\_sidebar} et \texttt{stack\_add\_page}}

Le véritable potentiel d'un \texttt{GtkStack} s'exprime lorsqu'il est couplé à un contrôleur externe. C'est ici qu'interviennent les deux helpers conçus par le module. 

La fonction \texttt{create\_stack\_sidebar} automatise la mise en place d'un panneau latéral de navigation. Son implémentation est redoutable de simplicité : elle instancie une \texttt{GtkStackSidebar} et la lie intimement à la pile passée en argument via \texttt{gtk\_stack\_sidebar\_set\_stack}. Dès cet appel, GTK assure une synchronisation bidirectionnelle automatique : les clics dans la barre changent les pages, et les changements de pages (via code) mettent à jour la sélection visuelle dans la barre, sans aucune ligne de logique additionnelle requise de la part du développeur.

La fonction \texttt{stack\_add\_page}, quant à elle, masque l'une des bizarreries de l'API C de GTK4. Historiquement, nommer une page et lui donner un titre lisible nécessitait de manipuler des objets intermédiaires obscurs comme le \texttt{GtkStackPage}. Notre helper contourne cette difficulté architecturale : il insère d'abord le widget enfant avec un identifiant système via \texttt{gtk\_stack\_add\_named}, puis, dans un mouvement fluide, emploie l'introspection objet \texttt{g\_object\_set} pour récupérer discrètement la page générée et forcer l'attribution de la propriété "title". Cela permet d'alimenter directement la \texttt{StackSidebar} évoquée précédemment avec des libellés propres, tout en maintenant le code de l'application cliente purement orienté widgets.

\begin{lstlisting}[language=C]
GtkWidget *create_stack_sidebar(GtkWidget *stack, const widget_style_config *style) {
    GtkWidget *sidebar = gtk_stack_sidebar_new();
    if (stack != NULL) {
        gtk_stack_sidebar_set_stack(GTK_STACK_SIDEBAR(sidebar), GTK_STACK(stack));
    }
    apply_widget_style(sidebar, style);
    return sidebar;
}

void stack_add_page(GtkWidget *stack, GtkWidget *child, const char *name, const char *title) {
    if (stack == NULL || child == NULL || name == NULL || title == NULL) {
        return;
    }

    gtk_stack_add_named(GTK_STACK(stack), child, name);
    g_object_set(gtk_stack_get_page(GTK_STACK(stack), child), "title", title, NULL);
}
\end{lstlisting}

\subsection{APIs GTK utilisées}

\begin{itemize}
    \item \textbf{\texttt{gtk\_grid\_new()}} : Instancie un conteneur en grille à deux dimensions permettant d'organiser les widgets selon des lignes et des colonnes. Elle est le point de départ de la fonction \texttt{create\_grid}, qui configure ensuite l'espacement et l'homogénéité.
    \item \textbf{\texttt{gtk\_box\_new()}} : Crée un conteneur linéaire simple qui dispose ses widgets enfants selon une orientation donnée. Le module l'utilise dans \texttt{create\_box} pour simplifier la mise en place de flux horizontaux ou verticaux.
    \item \textbf{\texttt{gtk\_stack\_new()}} : Prépare un conteneur multicouche affichant un seul widget à la fois parmi plusieurs pages. Cette API est au cœur de \texttt{create\_stack}, facilitant la gestion des transitions et de l'homogénéité des dimensions des pages.
    \item \textbf{\texttt{gtk\_stack\_sidebar\_new()}} : Génère une barre latérale de navigation compatible avec le widget \texttt{GtkStack}. Elle est utilisée par le helper correspondant pour fournir un moyen intuitif de changer de page dans une interface à onglets.
    \item \textbf{\texttt{gtk\_stack\_add\_named()}} : Insère un widget dans une pile en lui assignant un nom symbolique unique. Le wrapper l'utilise dans \texttt{stack\_add\_page} pour permettre un référencement aisé de chaque vue de l'application.
    \item \textbf{\texttt{g\_object\_set()}} : Modifie les propriétés d'un objet via une interface générique à base de chaînes de caractères. Dans le module, elle sert spécifiquement à définir le titre visible d'une page de pile, propriété portée par l'objet \texttt{GtkStackPage} interne à GTK.
\end{itemize}
