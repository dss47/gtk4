\section{Module \texttt{input.c}}

L'interaction utilisateur repose majoritairement sur la saisie de données. Ce module simplifie drastiquement la création de formulaires en masquant la configuration complexe des \texttt{GtkEntry}, des listes déroulantes (\texttt{GtkDropDown}), et des sélecteurs numériques. L'idée est de passer d'une logique procédurale ("créer un widget, puis lui ajouter un filtre numérique, puis définir sa limite") à une déclaration claire ("je veux un champ qui n'accepte que des nombres de 0 à 100").

\subsection{Fonction \texttt{create\_entry}}

L'entrée de texte, widget \texttt{GtkEntry}, constitue la pierre angulaire de toute application nécessitant des informations alphanumériques de la part de l'utilisateur. La complexité de cette implémentation, bien que camouflée derrière un prototype simple, gère de multiples protocoles d'accessibilité et de sécurité applicative.

Tout d'abord, la fonction adresse le problème récurrent des zones fantômes d'interaction : un champ vide est muet et perturbant. En injectant un éventuel \texttt{placeholder} via \texttt{gtk\_entry\_set\_placeholder\_text}, elle assure une indication sémantique persistante jusqu'à la première frappe de l'utilisateur. En parallèle, elle convertit dynamiquement la chaîne \texttt{purpose\_hint} (par exemple "url", "email" ou "number") via une fonction interne \texttt{parse\_input\_purpose} afin d'informer le gestionnaire d'affichage système du type de donnée attendu. Sur un environnement mobile, tactile, ou lors de l'utilisation d'un clavier virtuel On-Screen Keyboard (OSK), ce seul appel adapte le clavier contextuellement, prouvant la réactivité du composant aux contraintes multiplateformes.

Dans le registre de la sécurité logicielle, la gestion des mots de passe s'active grâce à un simple drapeau booléen. Lorsque la propriété \texttt{password} est validée, l'appel à \texttt{gtk\_entry\_set\_visibility} avec la valeur \texttt{FALSE} est émis. Cela obfusque immédiatement l'affichage des caractères en direct, le plus souvent remplacés par des points sécurisés, tout en préservant le stockage de la chaîne en mémoire tampon (buffer). Cette abstraction décharge le développeur des mécanismes manuels de masquage de chaînes. 

Enfin, la fonction intègre une limitation quantitative de saisie (le \texttt{max\_length}). C'est une prévention technique robuste face aux potentielles attaques par débordement de capacité (buffer overflow) sur la base de données qui récupérera les informations à l'autre bout de l'application, instaurant ainsi un contrat de sûreté dès l'Interface Utilisateur.

\begin{lstlisting}[language=C]
GtkWidget *create_entry(const entry_config *config) {
    GtkWidget *entry = gtk_entry_new();

    if (config == NULL) {
        return entry;
    }

    if (config->placeholder) {
        gtk_entry_set_placeholder_text(GTK_ENTRY(entry), config->placeholder);
    }
    if (config->default_text) {
        gtk_editable_set_text(GTK_EDITABLE(entry), config->default_text);
    }
    gtk_entry_set_input_purpose(GTK_ENTRY(entry), parse_input_purpose(config->purpose_hint));
    gtk_entry_set_visibility(GTK_ENTRY(entry), !config->password);
    gtk_editable_set_editable(GTK_EDITABLE(entry), config->editable);
    if (config->max_length > 0) {
        gtk_entry_set_max_length(GTK_ENTRY(entry), config->max_length);
    }
    apply_widget_style(entry, &config->style);

    if (config->on_changed) {
        g_signal_connect(entry, "changed", G_CALLBACK(config->on_changed), config->user_data);
    }

    return entry;
}
\end{lstlisting}

\paragraph{Macros associées :}
Ces macros de création directe agissent comme un véritable mini-langage dédié (DSL - Domain Specific Language) au-dessus de l'API C du wrapper. Elles exploitent les initialisateurs désignés en ligne pour figer définitivement certains champs sémantiques (comme \texttt{.password = true} pour un champ caché ou \texttt{.purpose\_hint = "email"} pour forcer un clavier spécifique sur mobile) tout en laissant d'autres paramètres (comme le \texttt{placeholder}) dynamiques et paramétrables par le développeur. Le résultat est une déclaration de l'interface utilisateur visuellement très hiérarchisée et presque auto-documentée, rappelant la fluidité des frameworks modernes déclaratifs (tels que SwiftUI, Flutter ou React), où la définition structurelle du widget traduit immédiatement son intention fonctionnelle sans noyer le lecteur dans le paramétrage bas niveau.

\begin{lstlisting}[language=C]
#define CREATE_PASSWORD_ENTRY(ph) create_entry(&(entry_config){.placeholder = ph, .password = true, .purpose_hint = "password"})
#define CREATE_NUMERIC_ENTRY(ph)  create_entry(&(entry_config){.placeholder = ph, .purpose_hint = "number"})
#define CREATE_EMAIL_ENTRY(ph)    create_entry(&(entry_config){.placeholder = ph, .purpose_hint = "email"})
#define CREATE_URL_ENTRY(ph)      create_entry(&(entry_config){.placeholder = ph, .purpose_hint = "url"})
\end{lstlisting}

\subsection{Fonction \texttt{create\_dropdown}}

Le vieux composant \texttt{GtkComboBoxText} s'avérait lourd, lent et nécessitait la gestion fastidieuse de modèles abstraits (les \texttt{GtkTreeModel}) même pour des listes de choix rudimentaires. Notre implémentation embrasse pleinement la transition vers la modernité offerte par GTK4 à travers le \texttt{GtkDropDown}, optimisant drastiquement le flux de travail.

La prouesse de \texttt{create\_dropdown} repose sur la fonction spécifique \texttt{gtk\_drop\_down\_new\_from\_strings}. En acceptant directement un simple tableau de chaînes en C à terminaison nulle (null-terminated char array) issu de la structure \texttt{dropdown\_config}, elle occulte entièrement la mécanique interne de création des \texttt{GtkStringList} et la tuyauterie des expressions de modèles d'arbres. Ce raccourci architectural autorise l'initialisation de listes d'options considérables en à peine quelques lignes de code pur, propulsant les performances d'instanciation de l'interface.

Dans une logique d'ergonomie avancée, la fonction offre une arme décisive pour naviguer dans de grands ensembles de données : l'activation conditionnelle de la recherche intégrée via \texttt{gtk\_drop\_down\_set\_enable\_search}. Si ce drapeau est actif, le wrapper greffe de manière transparente une zone de filtrage au-dessus du menu déroulant dès que l'utilisateur l'ouvre. Couplé au branchement direct d'un callback sur le signal "notify::selected" (qui avertit dynamiquement le programmeur des changements d'état par introspection d'objets GLib), ce composant se présente comme une interface de sélection haut de gamme, clef en main, pour un coût d'implémentation dérisoire.

\begin{lstlisting}[language=C]
GtkWidget *create_dropdown(const dropdown_config *config) {
    GtkWidget *dropdown;

    if (config == NULL || config->options == NULL) {
        return gtk_drop_down_new(NULL, NULL);
    }

    dropdown = gtk_drop_down_new_from_strings(config->options);
    gtk_drop_down_set_selected(GTK_DROP_DOWN(dropdown), config->selected_index);
    gtk_drop_down_set_enable_search(GTK_DROP_DOWN(dropdown), config->enable_search);
    apply_widget_style(dropdown, &config->style);

    if (config->on_selected) {
        g_signal_connect(dropdown, "notify::selected", G_CALLBACK(config->on_selected), config->user_data);
    }

    return dropdown;
}
\end{lstlisting}

\subsection{Fonction \texttt{create\_spin\_button}}

Un champ d'entrée standard (\texttt{GtkEntry}) est peu pertinent, voire risqué, pour gérer de la donnée exclusivement numérique nécessitant de l'incrémentation (comme un volume, une année, un âge ou une tarification). Le \texttt{GtkSpinButton} adresse cette problématique, mais son implémentation native souffre d'une complexité décourageante pour les néophytes, exigeant l'orchestration parallèle d'un modèle mathématique de comportement. Notre fonction \texttt{create\_spin\_button} unifie cette dualité en un seul composant cohérent.

Le cœur névralgique de ce widget n'est pas son rendu visuel, mais son moteur de règles, appelé le \texttt{GtkAdjustment}. Ce modèle défini via \texttt{gtk\_adjustment\_new} encapsule tous les paramètres mathématiques du composant dans une unique allocation mémoire : la valeur initiale, la limite plancher (\texttt{min\_value}), le plafond absolu (\texttt{max\_value}), la taille d'un saut mineur au clic d'une flèche (\texttt{step}), et la taille d'un saut majeur souvent lié aux touches PageUp/PageDown du clavier (\texttt{page}). En générant et en attachant automatiquement cet objet au widget visuel lors de l'appel à \texttt{gtk\_spin\_button\_new}, notre fonction épargne au développeur la corvée de gestion du cycle de vie des objets intermédiaires de la GLib.

Une fois la base logique arrimée, la fonction affine l'expérience utilisateur de manière granulaire. L'invocation de \texttt{gtk\_spin\_button\_set\_numeric} verrouille impitoyablement la frappe libre sur le clavier, rejetant tout caractère non valide à la source avant même la validation d'un formulaire, agissant comme un filtre (sanitizer) robuste. Enfin, si la propriété \texttt{wrap} est déclenchée via \texttt{gtk\_spin\_button\_set\_wrap}, elle confère au composant un comportement cyclique continu (atteindre le maximum et incrémenter relance le compteur au minimum, idéal pour le réglage d'horloges, comme le passage de 23h à 00h). De bout en bout, la fonction transforme un composant fragmenté en un outil de saisie hautement paramétrable et inaltérable.

\begin{lstlisting}[language=C]
GtkWidget *create_spin_button(const spin_button_config *config) {
    GtkAdjustment *adj;
    GtkWidget *spin;

    if (config == NULL) {
        adj = gtk_adjustment_new(0.0, 0.0, 100.0, 1.0, 5.0, 0.0);
        return gtk_spin_button_new(adj, 1.0, 0);
    }

    adj = gtk_adjustment_new(config->value, config->min_value, config->max_value, config->step, config->page, 0.0);
    spin = gtk_spin_button_new(adj, config->step, config->digits);
    gtk_spin_button_set_numeric(GTK_SPIN_BUTTON(spin), config->numeric_only);
    gtk_spin_button_set_wrap(GTK_SPIN_BUTTON(spin), config->wrap);
    apply_widget_style(spin, &config->style);

    if (config->on_value_changed) {
        g_signal_connect(spin, "value-changed", G_CALLBACK(config->on_value_changed), config->user_data);
    }

    return spin;
}
\end{lstlisting}

\paragraph{Macros associées :}
A l'instar des champs de texte, ces macros pré-configurent le comportement des \texttt{SpinButton} pour des cas d'usage typiques (pourcentages, âges, années). Elles masquent la complexité de paramétrage de l'objet \texttt{GtkAdjustment} sous-jacent (limites min/max, pas d'incrément, contraintes de décimales) derrière une syntaxe claire et univoque, augmentant considérablement la productivité lors du montage de formulaires de saisie complexes.

\begin{lstlisting}[language=C]
#define SPIN_PERCENT(v) (spin_button_config){.min_value = 0, .max_value = 100, .step = 1, .value = v, .digits = 0}
#define SPIN_AGE(v)     (spin_button_config){.min_value = 0, .max_value = 120, .step = 1, .value = v, .digits = 0}
#define SPIN_YEAR(v)    (spin_button_config){.min_value = 1900, .max_value = 2100, .step = 1, .value = v, .digits = 0}
\end{lstlisting}

\subsection{APIs GTK utilisées}

\begin{itemize}
    \item \textbf{\texttt{gtk\_entry\_set\_placeholder\_text()}} : Affiche un message d'incitation à l'intérieur du champ de saisie tant que celui-ci est vide. Le module l'utilise pour fournir des indications contextuelles à l'utilisateur sans encombrer l'interface avec des labels externes.
    \item \textbf{\texttt{gtk\_entry\_set\_input\_purpose()}} : Indique la nature des données attendues (URL, email, chiffres, etc.) pour optimiser le comportement des méthodes de saisie, comme le clavier virtuel sur mobile. Le wrapper l'utilise pour améliorer l'ergonomie des formulaires.
    \item \textbf{\texttt{gtk\_drop\_down\_new\_from\_strings()}} : Construit une liste déroulante moderne en utilisant un tableau de chaînes de caractères pour les options. Cette API simplifie grandement la création de menus de sélection dans le module par rapport aux anciens widgets GTK.
    \item \textbf{\texttt{gtk\_drop\_down\_set\_enable\_search()}} : Permet d'intégrer une barre de filtrage rapide à l'intérieur du menu déroulant. Le wrapper active cette fonctionnalité pour faciliter la sélection lorsque le nombre d'options devient important.
    \item \textbf{\texttt{gtk\_adjustment\_new()}} : Définit un modèle de données numérique spécifiant une valeur courante, des bornes minimale et maximale, ainsi que des pas de progression. Elle constitue la fondation logique du widget \texttt{GtkSpinButton} créé par le module.
    \item \textbf{\texttt{gtk\_spin\_button\_new()}} : Crée le widget de contrôle numérique lié à un ajustement. Elle permet à l'utilisateur de saisir ou d'incrémenter précisément une valeur numérique dans les limites définies par le wrapper.
\end{itemize}
