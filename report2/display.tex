\section{Module \texttt{display.c}}

Afficher de l'information statique ou animée sans interaction (labels, barres de progression, images) représente une grande partie de l'interface. Ce module encapsule ces éléments d'affichage pur. En abstrayant les nuances entre une simple image locale et une icône de thème système, le module offre une interface de création homogène, accélérant considérablement le rendu visuel.

\subsection{Fonction \texttt{create\_progress\_bar}}

La barre de progression est le vecteur par excellence du retour visuel d'une application (feedback). L'implémentation de \texttt{create\_progress\_bar} dans ce wrapper va au-delà de la simple définition d'une valeur chiffrée. Elle prend en charge une configuration à multiples facettes pour couvrir des scénarios d'utilisation variés.

Dès l'instanciation, la fonction applique la valeur courante (\texttt{fraction}), mais elle traite immédiatement les modificateurs d'affichage. La capacité à inverser le sens de remplissage (\texttt{gtk\_progress\_bar\_set\_inverted}) est primordiale pour les applications gérant des chronomètres à rebours ou s'adaptant aux standards culturels de lecture de droite à gauche (RTL). L'orientation, qu'elle soit horizontale ou verticale, est prise en compte via un casting sécurisé (\texttt{GTK\_ORIENTABLE}) montrant l'héritage objet du C dans GTK.

Cependant, le véritable apport de logique métier de cette fonction réside dans la gestion asynchrone émulée via le booléen \texttt{pulse\_once}. Lorsqu'une application lance une tâche dont la durée est inconnue (comme une requête réseau lointaine), spécifier une \texttt{fraction} est impossible. En appelant \texttt{gtk\_progress\_bar\_pulse}, la fonction bascule le widget dans un mode d'activité indéterminée, où un bloc visuel effectue des allers-retours continus pour signaler à l'utilisateur que le système n'est pas gelé, préservant ainsi l'ergonomie de l'expérience utilisateur globale.

\begin{lstlisting}[language=C]
GtkWidget *create_progress_bar(const progress_bar_config *config) {
    GtkWidget *progress = gtk_progress_bar_new();

    if (config == NULL) {
        return progress;
    }

    gtk_progress_bar_set_fraction(GTK_PROGRESS_BAR(progress), config->fraction);
    gtk_progress_bar_set_show_text(GTK_PROGRESS_BAR(progress), config->show_text);
    gtk_progress_bar_set_inverted(GTK_PROGRESS_BAR(progress), config->inverted);
    gtk_orientable_set_orientation(GTK_ORIENTABLE(progress), config->orientation);
    if (config->text != NULL) {
        gtk_progress_bar_set_text(GTK_PROGRESS_BAR(progress), config->text);
    }
    if (config->pulse_once) {
        gtk_progress_bar_pulse(GTK_PROGRESS_BAR(progress));
    }
    apply_widget_style(progress, &config->style);

    return progress;
}
\end{lstlisting}

\paragraph{Macros associées :}
Des macros de progression apportent une clarté sémantique sur les valeurs numériques flottantes.

\begin{lstlisting}[language=C]
#define PROGRESS_START 0.0
#define PROGRESS_HALF  0.5
#define PROGRESS_FULL  1.0
\end{lstlisting}

\subsection{Fonction \texttt{create\_spinner}}

Le \texttt{GtkSpinner} est l'alternative moderne et minimaliste à la barre de progression pour signifier une attente (l'équivalent de l'icône de chargement circulaire sur navigateur web). Bien que le widget en lui-même soit trivial, la fonction \texttt{create\_spinner} du wrapper comble une faille ergonomique courante de l'API GTK de base.

Dans GTK4, un \texttt{GtkSpinner} nouvellement créé est statique et invisible s'il ne possède pas une taille requise minimale. Notre implémentation intercepte ce problème en imposant une configuration dimensionnelle via \texttt{gtk\_widget\_set\_size\_request}, forçant le rendu d'un carré parfait si une taille est fournie dans la \texttt{spinner\_config}. 

De plus, l'état d'animation par défaut du widget natif est à l'arrêt. La fonction simplifie le lancement automatique en évaluant la propriété \texttt{spinning} de la configuration. Si le développeur l'exige, \texttt{gtk\_spinner\_start} est appelé lors de l'initialisation, permettant l'injection immédiate d'un indicateur de chargement fonctionnel, dynamique et dimensionné de manière appropriée au sein d'une mise en page complexe.

\begin{lstlisting}[language=C]
GtkWidget *create_spinner(const spinner_config *config) {
    GtkWidget *spinner = gtk_spinner_new();

    if (config != NULL) {
        if (config->size > 0) {
            gtk_widget_set_size_request(spinner, config->size, config->size);
        }
        if (config->spinning) {
            gtk_spinner_start(GTK_SPINNER(spinner));
        }
        apply_widget_style(spinner, &config->style);
    }

    return spinner;
}
\end{lstlisting}

\subsection{Fonction \texttt{create\_label}}

Bien que le composant de base par excellence paraisse inoffensif, le texte non maîtrisé est la cause numéro un des ruptures (cassures) d'interfaces graphiques (UI breakage). La fonction \texttt{create\_label} du module agit comme un bouclier architectural en offrant par défaut les outils de gestion du flux textuel.

Une fois le widget initialisé avec un potentiel texte nul pris en charge défensivement, la fonction active les attributs fondamentaux de typographie numérique. L'appel à \texttt{gtk\_label\_set\_selectable} garantit que les informations importantes, comme un hash cryptographique ou un message d'erreur, puissent être surlignées et copiées dans le presse-papiers par l'utilisateur sans nécessiter l'instanciation de champs d'édition (Entry) factices.

La vraie valeur ajoutée de la fonction réside dans son traitement anti-débordement. Plutôt que de laisser un texte interminable repousser les bordures de la fenêtre mère, la conjonction de \texttt{gtk\_label\_set\_wrap} (retour à la ligne automatique) et de \texttt{gtk\_label\_set\_ellipsize} (troncature par points de suspension, qu'elle soit au début, au centre ou à la fin selon la politique Pango choisie) sécurise la mise en page. Pour clore ce filet de sécurité géométrique, le paramètre \texttt{max\_width\_chars} force une coupure matérielle en nombre de caractères, prévenant toute dilatation anarchique d'un conteneur parent.

\begin{lstlisting}[language=C]
GtkWidget *create_label(const label_config *config) {
    GtkWidget *label = gtk_label_new(config && config->text ? config->text : "");

    if (config == NULL) {
        return label;
    }

    gtk_label_set_selectable(GTK_LABEL(label), config->selectable);
    gtk_label_set_wrap(GTK_LABEL(label), config->wrap);
    gtk_label_set_ellipsize(GTK_LABEL(label), config->ellipsize);
    if (config->max_width_chars > 0) {
        gtk_label_set_max_width_chars(GTK_LABEL(label), config->max_width_chars);
    }
    apply_widget_style(label, &config->style);

    return label;
}
\end{lstlisting}

\paragraph{Macros associées :}
Ces macros appliquent le concept de l'initialisation désignée et des littéraux composés pour standardiser la typographie. En injectant en ligne une sous-structure imbriquée \texttt{.style = \{.css\_class = "title-1", .margin\_bottom = 12\}}, la macro configure d'un seul trait le texte et son apparence (hiérarchie CSS et espacement d'aération). Cela évite au programmeur d'éparpiller des constantes de marges ou de noms de classes dans le code de composition de l'application cliente, garantissant ainsi que chaque sous-titre de l'application bénéficiera strictement du même rendu.

\begin{lstlisting}[language=C]
#define CREATE_TITLE(t)    create_label(&(label_config){.text = t, .style = {.css_class = "title-1", .margin_bottom = 12}})
#define CREATE_SUBTITLE(t) create_label(&(label_config){.text = t, .style = {.css_class = "title-2", .margin_bottom = 8}})
\end{lstlisting}

\subsection{Fonction \texttt{create\_image}}

L'affichage d'images dans GTK4 est historiquement fragmenté entre plusieurs widgets selon la source (fichiers locaux, surfaces Cairo en mémoire, icônes thématiques). La fonction \texttt{create\_image} résout cette fragmentation logicielle grâce au polymorphisme de retour (elle retourne un composant du type générique \texttt{GtkWidget*}) et en agissant comme une "Factory" interne qui choisit dynamiquement la bonne implémentation de bas niveau en fonction des champs configurés.

Si un chemin absolu ou relatif (\texttt{file\_path}) est fourni, la logique favorise l'instanciation de \texttt{GtkPicture}, le composant ultra-performant de GTK4 optimisé pour le redimensionnement fluide des bitmaps. En activant \texttt{gtk\_picture\_set\_can\_shrink}, on garantit que l'image locale peut descendre en dessous de sa taille native sans briser les contraintes de mise en page. S'ajoute à cela le contrôle crucial du ratio d'aspect via \texttt{gtk\_picture\_set\_content\_fit} qui choisit entre un zoom couvrant (FILL) ou un zoom conservateur (CONTAIN).

À l'inverse, si la configuration pointe vers une ressource symbolique nommée via \texttt{icon\_name} (ex: "folder-documents-symbolic"), la fonction dévie intelligemment vers l'instanciation d'un objet \texttt{GtkImage} classique, spécialement conçu pour interroger les répertoires de thèmes freedesktop.org au niveau du système d'exploitation. Elle assure également la conformité de l'affichage vectoriel de l'icône en forçant une conversion en taille de pixels stricte via \texttt{gtk\_image\_set\_pixel\_size}. Ce niveau d'abstraction permet au développeur d'application d'ignorer totalement la complexité des API de rendu graphique subjacentes.

\begin{lstlisting}[language=C]
GtkWidget *create_image(const image_config *config) {
    GtkWidget *widget;

    if (config == NULL) {
        return gtk_image_new();
    }

    if (config->file_path != NULL && config->file_path[0] != '\0') {
        GtkWidget *pic = gtk_picture_new_for_filename(config->file_path);
        gtk_picture_set_can_shrink(GTK_PICTURE(pic), TRUE);
        gtk_picture_set_content_fit(
            GTK_PICTURE(pic),
            config->keep_aspect ? GTK_CONTENT_FIT_CONTAIN : GTK_CONTENT_FIT_FILL
        );
        widget = pic;
    } else if (config->icon_name != NULL && config->icon_name[0] != '\0') {
        GtkWidget *img = gtk_image_new_from_icon_name(config->icon_name);
        if (config->pixel_size > 0) {
            gtk_image_set_pixel_size(GTK_IMAGE(img), config->pixel_size);
        }
        widget = img;
    } else {
        widget = gtk_image_new();
    }

    apply_widget_style(widget, &config->style);
    return widget;
}
\end{lstlisting}

\subsection{APIs GTK utilisées}

\begin{itemize}
    \item \textbf{\texttt{gtk\_progress\_bar\_set\_fraction()}} : Définit la fraction de remplissage de la barre de progression, acceptant une valeur flottante entre 0.0 et 1.0. Elle est utilisée dans \texttt{create\_progress\_bar} pour initialiser le niveau d'avancement visuel.
    \item \textbf{\texttt{gtk\_progress\_bar\_pulse()}} : Fait basculer la barre de progression en mode "activité indéterminée", déplaçant un bloc de manière cyclique. Le wrapper l'appelle pour indiquer que le programme est en cours de traitement sans pouvoir estimer le temps restant.
    \item \textbf{\texttt{gtk\_spinner\_start()}} : Démarre l'animation de rotation du widget spinner. Le module l'utilise pour activer visuellement l'indicateur de chargement dès sa création si la configuration le spécifie.
    \item \textbf{\texttt{gtk\_label\_set\_ellipsize()}} : Détermine le mode de troncature du texte (début, milieu ou fin) par des points de suspension lorsque l'espace est insuffisant. Elle permet au wrapper de garantir une mise en page propre même avec des labels imprévus.
    \item \textbf{\texttt{gtk\_picture\_new\_for\_filename()}} : Crée un widget d'image haute performance à partir d'un fichier local. Contrairement à \texttt{GtkImage}, cette API utilisée par le module facilite le redimensionnement proportionnel des images chargées.
    \item \textbf{\texttt{gtk\_image\_new\_from\_icon\_name()}} : Instancie un widget d'image affichant une icône issue du thème de l'utilisateur. Le wrapper l'exploite pour intégrer des symboles graphiques standards dans l'interface de manière simple et légère.
\end{itemize}
